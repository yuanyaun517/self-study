# 校园二手市场系统 - 技术文档

## 📊 一、系统概述

### 1.1 项目简介
**校园二手市场（School-Market）** 是一个面向高校学生的二手交易平台，提供商品发布、浏览、交易、社区论坛等功能。系统采用前后端分离架构，包含三个核心模块：

| 模块 | 技术栈 | 说明 |
|------|--------|------|
| **SecondMarketServer** | Spring Boot + MyBatis-Plus | 后端服务 |
| **SecondMarket** | 微信小程序 | 用户端 |
| **SecondMarketManager** | HTML + Layui + jQuery | 管理后台 |

### 1.2 技术架构
```
┌─────────────────────────────────────────────┐
│               前端展示层                      │
│  ┌──────────────┐      ──────────────┐     │
│  │  微信小程序端 │      │  管理后台Web │     │
│  └──────────────      └──────────────┘     │
└─────────────────────────────────────────────┘
                    ↕ HTTP/HTTPS
─────────────────────────────────────────────┐
│               后端服务层                      │
│  Spring Boot 2.4.5 + MyBatis-Plus 3.4.2    │
│  ├─ Controller 层 (RESTful API)             │
│  ├─ Service 层 (业务逻辑)                    │

```

## 🗄️ 二、数据库设计

### 2.1 ER 模型核心关系

```
user(用户) ──1:N── goods(商品)
   │                 │
   ├──1:N── address(地址)  ├──1:N── orders(订单)
   ├──1:N── cart(购物车)   │
   ├──1:N── collect(收藏)  │
   └──1:N── forum(论坛)    │
                           └──N:1── user(买家)

manager(管理员) ──1:N── banner(轮播图)
   │                   ├──1:N── notice(公告)
   │                   └──1:N── menu(菜单)
```

### 2.2 核心数据表

| 表名 | 主键 | 核心字段 | 说明 |
|------|------|----------|------|
| `user` | id | account, password, nickname, tel, balance, college | 用户表 |
| `goods` | id | name, price, type, send_user, manage | 商品表 |
| `orders` | id | userid, goodsid, price, state | 订单表 |
| `cart` | id | userid, goodsid, price | 购物车表 |
| `forum` | id | title, content, send_user | 论坛表 |
| `address` | id | userid, name, tel, detail | 地址表 |
| `manager` | id | account, password | 管理员表 |

### 2.3 关键索引设计

```sql
-- 商品表索引
INDEX idx_send_user(send_user)     -- 按发布者查询
INDEX idx_type(type)               -- 按分类查询
INDEX idx_manage(manage)           -- 按审核状态查询

-- 订单表索引
INDEX idx_userid(userid)           -- 用户订单查询
INDEX idx_state(state)             -- 按状态筛选

-- 收藏表唯一索引
UNIQUE INDEX uk_user_type_sid(user, type, sid)  -- 防止重复收藏
```

## 🔑 三、核心算法与业务逻辑

### 3.1 商品审核机制

**算法流程：**
```
用户发布商品 → manage=0(待审核)
                ↓
        管理员审核
        ↓        ↓
    通过(1)    不通过(2)
        ↓        ↓
    前台展示   用户可见/不可购买
```

**审核状态枚举：**
```
0 - 已发布（待审核）
1 - 审核通过
2 - 审核不通过
3 - 已删除
```

### 3.2 订单状态机

**状态转换：**
```
1(待付款) → 2(待发货) → 3(待收货) → 4(已完成)
    ↓
  5(已取消)
```

**业务规则：**
- 买家下单后状态为 `1`
- 卖家发货后变为 `2`
- 买家确认收货后变为 `3`
- 交易完成后变为 `4`
- 任何状态都可取消变为 `5`

### 3.3 购物车合并算法

**逻辑：**
```
// 相同商品只保留一条记录
UNIQUE INDEX uk_user_goods(userid, goodsid)

// 加入购物车时：
1. 检查是否已存在
2. 存在则更新价格
3. 不存在则新增记录
```

### 3.4 收藏去重机制

**唯一约束：**
```sql
UNIQUE INDEX uk_user_type_sid(user, type, sid)
```

**类型区分：**
- `type="goods"` - 商品收藏
- `type="forum"` - 论坛收藏

### 3.5 登录拦截过滤器

**过滤链逻辑：**
```
请求到达 → LoginCheckFilter
    ↓
检查白名单路径
    ↓ 是
直接放行
    ↓ 否
检查 Session
    ↓ 存在
放行 + 设置当前用户ID
    ↓ 不存在
返回 NOTLOGIN 错误
```

**白名单路径：**
```
"/employee/login",
"/user/sendMsg",
"/user/login",
"/user/register",
"/doc.html",
"/swagger-resources/**"
```

## 📡 四、API 接口文档

### 4.1 用户模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 用户注册 | POST | `/user/register` | 注册新用户 |
| 用户登录 | POST | `/user/login` | 账号密码登录 |
| 发送验证码 | GET | `/user/sendMsg` | 短信验证码 |
| 获取用户信息 | GET | `/user/{id}` | 查询用户详情 |
| 更新用户信息 | PUT | `/user` | 修改用户资料 |

### 4.2 商品模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 发布商品 | POST | `/goods` | 新增商品 |
| 商品列表 | GET | `/goods/list` | 分页查询 |
| 商品详情 | GET | `/goods/{id}` | 查看详情 |
| 按分类查询 | GET | `/goods/type/{type}` | 分类筛选 |
| 商品审核 | PUT | `/goods/manage` | 管理员审核 |

### 4.3 订单模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建订单 | POST | `/orders` | 下单 |
| 订单列表 | GET | `/orders/list` | 我的订单 |
| 订单详情 | GET | `/orders/{id}` | 查看详情 |
| 更新状态 | PUT | `/orders/state` | 发货/收货 |

### 4.4 购物车模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 添加商品 | POST | `/cart` | 加入购物车 |
| 购物车列表 | GET | `/cart/list` | 查看购物车 |
| 删除商品 | DELETE | `/cart/{id}` | 移除商品 |
